# Practical 04: Windows Log Analysis & Failed Login Investigation

## Overview

This practical focuses on understanding how Windows records authentication and security-related events using the built-in Event Viewer utility.

The goal of this exercise was to explore Windows Security Logs, locate failed login attempts, review authentication event details, and understand the basic investigation process followed by security analysts during authentication-related investigations.

The practical was performed inside a Windows 10 virtual machine using Event Viewer. To keep the environment safe and stable, no system settings, registry values, security policies, or audit configurations were modified. The exercise focused entirely on reviewing and analyzing recorded security events.

This practical helps build foundational knowledge useful for IT Support, System Administration, SOC Operations, Blue Teaming, Incident Response, and Cybersecurity roles.

# Part 1: Introduction to Windows Security Logs

## Objective

To explore Event Viewer and understand how Windows organizes and stores system and security-related events.

## Topics Covered

* Event Viewer Overview
* Windows Logs
* Application Logs
* Security Logs
* Setup Logs
* System Logs
* Authentication Monitoring

## What Was Observed

Event Viewer was opened from the Start Menu inside the Windows 10 virtual machine.

The Windows Logs section was expanded to review the different categories used by Windows to store event information.

The following log categories were observed:

| Log Category | Purpose                                      |
| ------------ | -------------------------------------------- |
| Application  | Events generated by installed applications   |
| Security     | Authentication and security-related events   |
| Setup        | Installation and configuration events        |
| System       | Operating system and hardware-related events |

Among these categories, the Security Log was identified as the primary source for authentication investigations because it records login activity and other security-related events.

* Event Viewer Navigation
* Windows Log Exploration
* Security Log Identification
* Authentication Monitoring Fundamentals

# Part 2: Finding Failed Login Attempts

## Objective

To locate failed authentication attempts within the Windows Security Log.

## Topics Covered

* Security Log Analysis
* Event IDs
* Event ID 4625
* Log Filtering
* Authentication Failures

## What Was Observed

The Security Log was reviewed and filtered using Event ID 4625.

Event ID 4625 records failed logon attempts and is commonly reviewed during authentication-related investigations.

After applying the filter, multiple failed login events were identified and reviewed.

The filtering process helped isolate authentication failures from the large number of events recorded within the Security Log.

The following information was reviewed:

| Field         | Description                                          |
| ------------- | ---------------------------------------------------- |
| Time Created  | Time when the failed login occurred                  |
| Event ID      | Unique identifier associated with the event          |
| Task Category | Category associated with the authentication activity |
| Keywords      | Indicates the event was recorded as an Audit Failure |

* Event Filtering
* Security Log Analysis
* Authentication Event Identification
* Event Review Techniques

# Part 3: SOC-Level Investigation

## Objective

To understand how security analysts review failed login events and identify activity that may require further investigation.

## Topics Covered

* Event Details Analysis
* Authentication Investigation
* Account Review
* Failure Analysis
* Pattern Recognition
* Security Monitoring Concepts

## What Was Observed

After locating Event ID 4625 entries, the event details were examined to better understand the authentication failures.

Several important fields were reviewed within each event.

| Field              | Purpose                                                |
| ------------------ | ------------------------------------------------------ |
| Account Name       | Identifies the user account involved                   |
| Logon Type         | Indicates how the authentication attempt was performed |
| Failure Reason     | Explains why the login attempt failed                  |
| Source Information | Provides additional context about the event            |

Multiple Event ID 4625 entries were reviewed to understand how analysts investigate authentication-related activity and identify patterns across events.

## Indicators Reviewed

The following indicators were discussed as examples of activity that may require additional review:

| Indicator                  | Why It May Require Review                                        |
| -------------------------- | ---------------------------------------------------------------- |
| Repeated Failed Logins     | May indicate password guessing or repeated authentication issues |
| Multiple Accounts Targeted | Could suggest broader authentication-related activity            |
| Unusual Login Activity     | Authentication attempts occurring outside normal usage patterns  |
| High Failure Volume        | Large numbers of failures occurring within a short period        |

It was also discussed that failed login events do not automatically indicate malicious activity. Authentication failures can occur for legitimate reasons such as incorrect passwords or user mistakes.

## Skills Developed

* Authentication Investigation
* Event Detail Analysis
* Pattern Recognition
* Security Monitoring Awareness

# Part 4: Investigation Summary & SOC Workflow

## Objective

To understand the basic workflow used during authentication-related investigations.

## Topics Covered

* Investigation Workflow
* Security Event Review
* Authentication Analysis
* Event Correlation Concepts
* Incident Investigation Fundamentals

## What Was Discussed

The practical concluded with a review of the investigation process followed throughout the exercise.

The workflow used during the practical included:

1. Opening Event Viewer.
2. Accessing Windows Security Logs.
3. Filtering events using Event ID 4625.
4. Reviewing authentication event details.
5. Comparing multiple failed login events.
6. Looking for recurring patterns.
7. Determining whether additional investigation may be required.

A simple real-world scenario was also discussed where a user reports being unable to log into a Windows system. Event Viewer and Security Logs were used as the starting point for reviewing authentication-related activity.

## Skills Developed

* Investigation Workflow Understanding
* Authentication Troubleshooting Concepts
* Basic SOC Investigation Methodology
